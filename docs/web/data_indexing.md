## Web App Data Indexing

Canonical source for how the web app discovers players and outings.

### dataIndex.ts

`web/lib/dataIndex.ts` is the single source of truth for which players and outings appear in the UI.

### Player model

```typescript
interface Player {
  id: string;        // e.g. "DJames1"
  name: string;      // e.g. "Dillon James"
  throws: "R" | "L"; // from Arsenals.csv pitcher_hand column
  outings: Outing[];
}
```

### Outing model

```typescript
interface Outing {
  id: string;        // e.g. "DJames1/2025_03_26"
  label: string;     // e.g. "Mar 26, 2025 – James (54 pitches)"
  csvPath: string;
  overlayDir: string;
  clipsDir: string;
}
```

Paths are generated by `buildDataPaths(playerId, dateId)`.

### Adding a new player or outing

See `docs/runbooks/publish_outing.md` for the full workflow.

### Outing selection logic

1. If `outingId` query parameter is present, use that outing.
2. Otherwise, use the first outing in the player's `outings` array.
3. If no outings exist, the page returns 404.

Query string format: `/player/<playerId>?outingId=<playerId>/<dateId>`

The `OutingSelect` component updates the URL on dropdown change.

### Importing boxscore stats

Use the Sidearm boxscore importer to write static JSON game logs into `web/public/stats/` so the web app can read them.

```bash
python3 scripts/import_boxscore.py --url "https://babsonathletics.com/sports/baseball/stats/2025/suffolk/boxscore/14570" --player "Chase Burrows" --team babson
```

Run tests locally:

```bash
pytest -q
```

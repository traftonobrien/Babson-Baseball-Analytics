# Phase 8 Plan 02: Integrate Engine Guardrails Into Live Workflow

<objective>
Use the new engine state to make the charting workflow footproof: valid closeout buttons, clearer next-action guidance, and cleaner pitcher-segment transitions without breaking the top-canvas / bottom-dock layout baseline.
</objective>

## Objective
- **What:** Wire the deterministic engine into the live charting UI and segment workflow.
- **Why:** A correct engine is not enough if the UI still encourages invalid sequencing or leaves the scorer guessing about the next step.
- **Output:** A guardrailed workflow that keeps charting operators on a valid path.

# Phase 8 Plan 01: Build Deterministic Charting Engine Rules

<objective>
Replace demo-grade count and outs reconstruction with a deterministic rules layer that can derive the live charting state from persisted pitches, plate appearances, and pitcher segments.
</objective>

## Objective
- **What:** Add an engine-level ruleset for pitch progression, PA closeout semantics, outs recorded, inning rollover, batter advancement, and open-PA state reconstruction.
- **Why:** The app cannot be trusted in live use while `GameStore` relies on partial string checks and mutable counters that drift from saved data.
- **Output:** Pure rules embedded in the iPad charting model layer and integrated into `GameStore`.

# Phase 12.1: Live AB player profile integration - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning
**Source:** PRD Express Path (.planning/phases/12.1-live-ab-player-profile-integration/12.1-PRD.md)

<domain>
## Phase Boundary

This phase integrates Live AB analytics into the existing player profile experience.

It should:
- add a new `Live AB` tab to the current player profile tabs,
- support both pitcher and hitter Live AB data from player pages,
- surface compact player-level overviews and recent charted-session history,
- reuse the shared charting analytics layer instead of creating duplicate formulas.

It should not:
- replace the global Live AB leaderboard,
- redesign the charting session overview page,
- trigger a broader non-charting player-profile rewrite.

</domain>

<decisions>
## Implementation Decisions

### Locked Decisions
- Add `Live AB` as a new tab inside the existing player profile tab system rather than creating a separate profile route family.
- Support both pitcher and hitter Live AB views from player pages.
- The Live AB tab must work for players with pitcher-only data, hitter-only data, or both.
- Emphasize at-a-glance player overviews plus session history rather than copying the global leaderboard table into the profile.
- Reuse the shared charting analytics layer and canonical player identity utilities instead of introducing page-local formulas.
- Preserve the current player-profile visual language and tab chrome.
- Empty states must be explicit when a player has no Live AB data for one or both roles.

### Claude's Discretion
- Whether the Live AB tab uses an internal role switcher for two-way players or stacks pitcher and hitter sections vertically.
- Exact stat-card composition, section ordering, and responsive layout inside the Live AB tab.
- Whether to add lightweight navigation links into charting session detail pages, filtered leaderboard views, or both.
- How much the existing profile hero should become role-aware in this phase versus staying mostly unchanged.

</decisions>

<specifics>
## Specific Ideas

- `web/app/players/[slug]/PlayerProfileTabs.tsx` is the natural insertion point for the new tab.
- `web/app/players/[slug]/page.tsx` currently fetches season pitching leaderboard data only, even though the file already defines batting metric helpers.
- The current player profile hero explicitly renders `Pitching Profile`, so planning should decide how hitter-capable profiles are represented.
- `/charting/leaderboard` and `web/lib/charting/analytics.ts` already establish the shared cross-session charting metric contract.
- Requirement anchor: `PORT-05` (`Charted games appear directly in player profile tabs and other portal hubs`).

</specifics>

<deferred>
## Deferred Ideas

- Broader redesign of the game session overview page (existing Phase 13 roadmap work)
- Replacing the global Live AB leaderboard with profile pages
- Non-charting overhauls to the player profile system outside the new Live AB tab and required role-awareness support

</deferred>

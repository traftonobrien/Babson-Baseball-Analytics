# Phase 12.1 PRD: Live AB Player Profile Integration

**Date:** 2026-03-10
**Requirement Anchor:** PORT-05
**Milestone:** v2.0 Live AB Analytics

## Objective

Bring Live AB analytics into the existing player profile experience so coaches can evaluate charted pitcher and hitter performance from each player page, not only from the global leaderboard.

## Problem

The repo now has:
- a shared charting analytics engine,
- per-game pitcher and hitter breakdowns,
- a global Live AB leaderboard,

but the player profiles still stop at Overview, Trackman, Command, and Mechanics. The profile route also still loads season pitching data only, even though the file already contains batting metric definitions. That leaves Live AB insights disconnected from the main player-development surface.

## Locked Decisions

1. Add **Live AB** as a new tab inside the existing player profile tab system rather than creating a separate profile route family.
2. Support **both pitcher and hitter Live AB views** from player pages.
3. The Live AB tab must work for players with:
   - pitcher-only Live AB data,
   - hitter-only Live AB data,
   - both pitcher and hitter data.
4. The new tab should emphasize **at-a-glance player overviews plus session history**, not just duplicate the global leaderboard table.
5. Reuse the shared charting analytics layer and canonical player identity utilities instead of introducing page-local formulas.
6. Preserve the current player-profile visual language and tab chrome; this should feel like a first-class profile tab, not a separate app embedded inside the page.
7. Empty states must be explicit when a player has no Live AB data for one or both roles.

## User Stories

- As a coach opening a pitcher profile, I want to see that pitcher's Live AB workload, core rates, and recent charted sessions in one tab.
- As a coach opening a hitter profile, I want to see that hitter's Live AB approach and production metrics in one tab.
- As a coach comparing a two-way player, I want the same profile page to surface both Live AB pitching and hitting views without needing a separate identity or route.
- As a staff user navigating from the players hub, I want the profile page to remain the canonical place where Trackman, Command, Mechanics, season stats, and Live AB analytics come together.

## Acceptance Criteria

1. The player profile tab strip exposes a **Live AB** tab.
2. On pitcher-capable players, the Live AB tab shows a pitcher-oriented overview built from cross-session charting data.
3. On hitter-capable players, the Live AB tab shows a hitter-oriented overview built from cross-session charting data.
4. If a player has both datasets, the Live AB tab presents both cleanly without forcing a second route.
5. The Live AB tab includes session-history access so a coach can move from profile summary into charted-session detail.
6. The profile route no longer assumes a pitching-only data model when the player should be represented as a hitter.
7. All Live AB metrics are sourced from the shared charting analytics contracts.
8. No-data states are explicit and role-aware.

## Implementation Context

### Existing Surfaces

- `web/app/players/[slug]/page.tsx`
  - currently fetches the pitching leaderboard for profile season stats,
  - renders a `Pitching Profile` hero pill,
  - passes data into `PlayerProfileTabs`.
- `web/app/players/[slug]/PlayerProfileTabs.tsx`
  - current tabs: `Overview`, `Trackman`, `Command`, `Mechanics`,
  - already uses the shared leaderboard chrome and profile-specific hub links,
  - is the natural insertion point for a new `Live AB` tab.
- `web/lib/charting/analytics.ts`
  - already supports aggregated pitcher and hitter Live AB metrics.
- `/charting/leaderboard`
  - already proves the Live AB cross-session aggregation path and metric set.

### Design Direction

- The Live AB tab should read like a compact player-development overview:
  - role-aware summary cards,
  - recent-session list or history rail,
  - quick links into deeper charting pages where useful.
- Avoid copying the full leaderboard table into the profile.
- Keep the tab usable on both desktop and narrower widths.

## Phase Boundary

This phase is about **profile integration and per-player Live AB presentation**.

In scope:
- adding the new tab,
- shaping the profile-level charting data,
- handling pitcher vs hitter vs two-way presentation,
- surfacing recent sessions and overview metrics.

Out of scope:
- redesigning the session overview page,
- replacing the global leaderboard,
- broad non-charting player-profile redesign.

## Open Product Questions To Resolve In Planning

- Should the Live AB tab use one internal role switcher when a player has both pitcher and hitter data, or stack both sections in one scrollable tab?
- Should the profile hero remain `Pitching Profile` for all players, or become role-aware once hitter usage is supported?
- Which session links should be primary from the tab:
  - charting game detail,
  - filtered leaderboard/drill-down,
  - or both?

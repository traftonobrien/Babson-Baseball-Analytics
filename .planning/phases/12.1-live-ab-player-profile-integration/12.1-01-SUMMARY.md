# Phase 12.1 Plan 01 Summary

**Added a shared Live AB player-profile helper that resolves pitcher and hitter charting data through the existing analytics and canonical identity layers**

## Accomplishments
- Added `web/lib/charting/playerProfile.ts` to centralize Live AB player-profile loading for pitcher-only, hitter-only, two-way, and no-data players.
- Reused `computePitcherAggregation`, `computeHitterAggregation`, `countPitcherInnings`, `getCanonicalName`, and `getCanonicalPlayerId` instead of re-implementing profile-local stat math.
- Added `web/lib/charting/playerProfile.test.ts` with coverage for pitcher identity resolution, hitter alias merging, two-way availability, and explicit no-data behavior.
- Verified `npm --prefix web test -- --run lib/charting/playerProfile.test.ts` passes.

## Decisions Made
- Kept the profile contract display-ready so `/players/[slug]` can consume one typed object instead of stitching together DB rows in the route.
- Resolved hitter identity by canonical player ID first and normalized alias candidates second because charted hitter names are stored as raw text rather than canonical IDs.
- Counted pitcher innings through the shared inning-span helper so profile totals stay aligned with the Live AB leaderboard/session-overview fixes.

## Next Phase Readiness
- Live AB player-profile data is now centralized in one helper contract.
- The next step is wiring that contract into the player-profile tab shell and rendering pitcher/hitter overview panels from it.

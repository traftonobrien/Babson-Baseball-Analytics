# Phase 08.1 Plan 02: Add Live AB and Game Mode Workflow

<objective>
Introduce an explicit mode model so the app can support practice/live-AB charting and real game charting without overloading one workflow with the other's requirements.
</objective>

## Objective
- **What:** Add a charting mode concept, a pre-AB setup flow for `Live AB`, and controlled correction affordances for `Game` mode.
- **Why:** The product needs to support both practice scenarios with manual context and lineup-driven game scoring without collapsing into a fully manual screen.
- **Output:** A mode-aware charting workflow that separates controlled practice entry from real-game flow.

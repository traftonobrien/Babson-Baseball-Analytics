1 — Product Overview
Pitch Tracker measures pitching command by comparing where a catcher sets up (glove target) to where the ball actually arrives, using a center-field camera. It combines SAM 2 computer vision, MediaPipe biomechanics, Trackman radar data, and live game stats into a single web portal for Babson Baseball.

2 — Command Pipeline (SAM 2)
Core Vision Pipeline
The primary pipeline measures pitch command from video. A static center-field camera captures each pitch. The operator marks target and arrival frames, SAM 2 segments the glove and ball, and the system computes miss distance in both pixels and inches.

SAM 2 Segmentation
Image predictor (fast path) segments glove mask at target frame and ball mask at arrival frame. All detection within a cropped ROI bounding the catcher area. Debug mode uses video predictor for multi-frame propagation.
Miss Calculation
Miss vector: dx = ball_x - target_x, dy = ball_y - target_y. Arm-side/glove-side labeling is relative to pitcher handedness. Pixel distance converted to inches via calibration constant.
Scrubber UI
PyQt frame scrubber for marking target and arrival frames. Outputs clips and pitch_log.json. At 30fps, a 90mph fastball moves ~17" per frame. Manual ball click required.
Outputs
Per-pitch: overlay video, CSV row with miss distance + direction + zone classification. Per-outing: aggregated pitch_data_overlay_lite.csv ready for web.
3 — Mechanics CV Pipeline
Biomechanics Analysis
A secondary pipeline analyzes pitcher mechanics from open-side or front-view video. MediaPipe Pose estimates body keypoints, heuristic algorithms detect delivery phases, and 7 Mustard-style benchmarks score the motion. The coach pack bundles annotated frames, video cliplets, and coaching callouts.

Coach Pack Output
4 annotated key frames (full-res with skeleton, badge bar, failure callouts). A 1x4 horizontal strip. 3 video cliplets (set-to-FS, FS-to-release, release+15). notes.json with metric values and coaching strings.
Annotations
Skeleton overlay, trunk lines, balance zone, swivel zone, head path, hip drift path, angle badges, and priority coaching cues. All scale relative to 720p reference. Supports slow-mo and hold-review video generation.

6 — Web Application
Next.js Portal
The web app is the user-facing product. Built with Next.js 16, React 19, TypeScript, and Tailwind CSS v4. Deployed on Vercel. All data is static JSON/CSV served from web/public/data/. No database, no server-side API.

Auth
Cookie-based team login. Single shared password. Protects all player data routes. Public landing page only.
Data Pattern
Static JSON + CSV at build time. dataIndex.ts registers players and outings. No runtime API. Git push triggers Vercel deploy.

7 — Data Architecture
Folder Contracts
Two mirrored directory trees form the backbone. Processing writes to local outings/, publishing copies to web/public/data/. Player ID format: DJames1. Date format: yyyy_mm_dd.
Local (Processing)
outings/<playerId>/<dateId>/

Contains: clips/ (raw pitch clips + pitch_log.json), results/ (overlay videos, masks), pitch_data_overlay_lite.csv
Web (Published)
web/public/data/<playerId>/<dateId>/

Contains: clips, overlays, CSV. Registered in dataIndex.ts via buildDataPaths(playerId, dateId)
Mechanics Output
output/mechanics/<slug>/<clip_slug>/

Contains: benchmarks.json, phases.json, metrics.json, coach_pack/ (PNGs, strip, cliplets, notes.json)
Key Data Files
config.yaml — ROI, calibration
data/Arsenals.csv — pitch type arsenals
web/lib/dataIndex.ts — player/outing registry
web/lib/handedness.ts — direction normalization
Invariant: Miss vector is always dx = ball_x - target_x, dy = ball_y - target_y. Arm-side/glove-side labels derive from dx * arm_sign. Camera is always center-field, behind pitcher, facing home plate. These are non-negotiable.
